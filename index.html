프로젝트 목표

현재 AI 기반 효과음 생성 기술은 영상·미디어와의 높은 종속성을 충분히 반영하지 못해 실무 활용에 한계가 있다. 효과음은 단독 품질이 아닌 싱크로나이징, 믹싱, 맥락 적합성이 동시에 충족되어야 하며, 이는 생성 중심 AI 구조만으로는 해결할 수 없다.

본 프로젝트는 AI 기반 생성 중심 구조에서 벗어나, 영상 분석 기반 자동 사운드 배치 시스템을 통해 실무 환경에 적합한 효과음 워크플로우를 구현하는 것을 목표로 한다. 사운드 에셋 수급은 이미 시장에 검증된 라이브러리를 활용하고, AI는 싱크로나이징과 배치 자동화를 담당하는 분석·매칭·배치 중심 구조를 지향한다.

1차 목표는 에디팅 단계의 자동화다 — 적절한 사운드 에셋 판별, 영상 싱크로나이징, 기초 편집. 오디오 믹스 단계는 사운드 에디팅이 완료된, 즉 미디어의 모든 요소가 커버된 이후에 우선순위와 연출을 고도화하는 작업이다. 따라서 믹싱은 에디팅이 일정 수준에 도달한 후 접근해야 할 영역이며, 이 구조는 실제 정규 사운드 디자인 워크플로우와 동일하다.

본 프로젝트는 현재 진행 중인 R&D 단계로, 프로덕션 수준의 완성도를 보장하지 않습니다.

현재 구현 범위: 영상 입력 → Claude API 기반 씬·캐릭터·재질 분석 → 이벤트 타임라인 JSON 출력 → 타임라인 뷰어 배치까지의 파이프라인이 구현되어 있습니다. 단, 실제 오디오 에셋 적용 단계는 포함되지 않으며, Foley 및 Ambience 타임라인 데이터는 분석 정확도 고도화 작업이 진행 중입니다.

핵심 인사이트
AI와 인간의 미디어 분석 방식 차이에서 도출한 실무적 방향

이 프로젝트를 통해 확인한 가장 실질적인 발견은, AI가 무엇을 잘하는지가 아니라 어떤 구조의 문제를 잘 푸는지, 그리고 AI가 미디어를 인간과 전혀 다른 방식으로 분석한다는 사실 자체에서 오는 가능성과 한계에 관한 것이다. AI는 인간이 당연하게 여기는 맥락을 놓치는 반면, 인간이 직관적으로 처리하는 시각 정보를 비구조적으로 분해해서 인간보다 정확하게 분류하는 영역도 존재한다. 어느 쪽이 더 우월하다는 게 아니라, 이 차이를 명확히 이해하는 것 자체가 AI를 실무에 통합하는 데 있어 핵심적인 출발점이었다.

Claude API는 씬 안에서 일어나는 것들 — 공간의 성격, 재질, 캐릭터 식별, 명확한 물리적 동작 — 을 이미지 기반으로 매우 정확하게 분류해낸다. HAND, CLOTH, Ambience처럼 "무엇이 존재하는가(existence)"를 판단하는 문제에서는 실무 적용 가능한 수준의 결과를 일관되게 냈다.

반면 Footstep 싱크는 다른 구조의 문제였다. 발이 지면에 닿는 순간은 "무언가가 있는가"가 아니라 "움직임이 언제 멈추는가(motion termination)"의 문제다. 이것은 프레임 간 연속적인 수치 비교를 요구하는데, 현재 LLM 비전 구조는 이미지를 개별적으로 처리하기 때문에 이 판단을 구조적으로 수행하기 어렵다. 7가지 접근(픽셀 차분, Pose Estimation, 프롬프트 고도화, ffmpeg 파이프라인, OpenCV 3신호 추적 등)을 시도했으나 모두 같은 벽에 부딪혔다.

결론적으로: AI가 잘 풀 수 있는 문제와 현재 구조에서 한계가 있는 문제를 명확히 구분하고, 잘 풀 수 있는 영역부터 완성도를 높이는 것이 현실적인 방향이다. Ambience와 SFX-Hard EFX는 전자에 해당하고, Footstep 정밀 싱크는 현재 개인 단위 프로젝트의 범위를 넘어서는 문제일 수 있다.

▶1. 문제 정의Problem Definition

a. 미디어 종속성

오디오는 영화, 게임, 광고 등 시각 기반 미디어에 대한 의존성이 매우 높은 매체이다. 음악을 제외하면, 미디어와 분리된 상태로 독립적인 상품성을 갖는 오디오는 매우 제한적이다.

특히 효과음은 미디어 의존성이 가장 높은 영역으로, 단독으로는 완성된 상품이 되기 어렵다. 사운드 믹싱 과정에서는 일반적으로 다이얼로그, 음악, 효과음, 폴리, 앰비언스 순으로 가시성을 조절하며, 하위 요소로 갈수록 '티 나지 않게' 배치되는 경우가 많다. 즉, 개별 효과음의 화려함보다 자연스럽고 이질감 없는 연출이 우선적으로 요구된다. 아무리 완성도가 높은 효과음이라 하더라도, 영상 맥락과 어긋날 경우 이는 명백한 오류로 인식된다.

따라서 효과음 설계의 핵심은 미디어에 자연스럽게 융합되는 것이다. 이를 위해서는 다음과 같은 최소 기준이 충족되어야 한다.

1. 싱크로나이징 (Sound Edit)
  • 영상 기준 프레임 단위 동기화
  • 이벤트 및 시퀀스에 대한 정확한 반응
  • 인터랙티브 콘텐츠 환경 대응
2. 믹스 및 마스터링 (Sound Mix)
  • 다이얼로그, 음악, 효과음, 폴리, 앰비언스 간 볼륨 밸런스 유지
  • 통합 dB LUFS, dBTP 기준 충족
  • SFX 개별 볼륨 관리 및 헤드룸 확보
  • 대사 및 음악과의 가시성 조절
3. 음질 관리 (Edit)
  • 클리핑, 클릭, 크랙클, 배경 노이즈 제거
  • 깨끗한 파형 유지
4. 에디팅 완성도 (Edit)
  • 질감과 무게감의 적절한 표현
  • 장면 맥락에 맞는 다이나믹 구성

b. AI 효과음 생성 기술의 한계

AI 기반 효과음 생성 기술은 음질 관리 및 에디팅 완성도 측면에서 일정 부분 기여할 수 있다. 이는 오디오 파일 자체의 품질만으로 평가 가능한 영역에 해당한다. 그러나 해당 방식에는 다음과 같은 구조적 한계가 존재한다.

비용 구조의 한계

고품질 효과음 에셋은 이미 시장에 충분히 축적되어 있으며, 이를 구매·활용하는 방식이 비용 측면에서 효율적인 경우가 많다. 반면, 생성형 AI 방식은 호출당 GPU 연산 자원을 지속적으로 소모하므로 장기적으로 운영 비용 부담이 증가한다.

품질 통제의 한계

현행 생성형 효과음 모델은 음질 관리 및 정밀한 에디팅 완성도를 안정적으로 확보하기 어렵다. 생성 과정에서 발생하는 클리핑(Clipping) 및 질감의 불균형 등은 필연적으로 수동 후반 보정 공정을 동반하며, 이는 결과적으로 '자동화'를 통한 생산성 향상이라는 본질적 이점을 상쇄시킨다.

또한, 생성형 모델의 특성상 동일한 입력값에 대해 일관된 레이어 구조나 위상(Phase)을 보장할 수 없다는 비결정론적 변칙성은, 정교한 믹싱과 사운드 레이어링이 필수적인 전문가용 파이프라인에 통합하기 어려운 제약 요인으로 작용한다.

미래 전략적 관점: 단, 이러한 품질 통제의 한계는 멀지 않은 미래에 기술적으로 해결될 영역임을 전제해야 한다. 기술의 성숙도가 상향 평준화되는 시점에는 '누가 더 잘 만드는가'보다 '누가 더 정확한 맥락(Context)에 배치하는가'의 싸움이 될 것이다. 따라서 현행 모델의 불완전성을 인지하는 동시에, 생성 결과물의 변칙성을 제어하고 영상 맥락에 즉각 대응할 수 있는 분석·배치 중심의 워크플로우를 선제적으로 구축하여 기술적 선점 효과를 극대화하고자 한다.

활용성 및 편의성의 한계

현재 시장에는 이미 다양한 고품질 사운드 디자인 에셋이 존재한다. 이러한 환경에서 생성형 AI가 경쟁력을 확보하기 위해서는 단순 생성이 아닌 입력 반응성 및 작업 효율 향상에서 우위를 확보해야 한다. 그러나 실제로는 다음과 같은 문제가 발생한다.

  • 전문가 사용자는 이미 고유의 워크플로우와 노하우를 구축하고 있어 생성 기반 도구의 활용도가 낮다.
  • 비전문 사용자는 자연어 기반 사운드 지시 능력이 제한적이어서 원하는 결과를 안정적으로 도출하기 어렵다.
  • 결과적으로 입력 대비 출력의 일관성과 신뢰도가 낮아 실무 환경에서 반복 활용이 어렵다.
비즈니스 구조의 한계 — 생산·결정·책임 3단계

콘텐츠 기반 서비스는 일반적으로 '생산–결정–책임'의 3단계 구조를 따른다. AI 기반 효과음 생성 서비스 또한 동일한 구조 내에서 평가할 수 있다.

  • 생산 단계 (AI 효과음 생성): AI를 통해 효과음을 생성하며, 이는 사운드 제작 과정 중 가장 초기 단계인 '에셋 확보'에 해당한다. 생성 기술은 사운드 디자인의 전체 과정이 아닌, 재료 수급 단계에 국한된 기능을 수행한다. 현행 생성 기술은 클리핑, 노이즈, 질감 불균형, 다이내믹 제어 미흡 등 세부 품질 관리 요소에서 한계가 존재하며, 생성된 에셋은 여전히 싱크 조정, 레벨 보정, 레이어 구성 등 후속 작업을 요구한다.
  • 결정 단계 (AI 효과음 구매 및 활용 판단): 전문가 집단(결정 역량 보유)은 기존 워크플로우 대비 품질·효율·비용 측면의 우위를 확인하지 못할 경우 AI 생성물을 활용할 유인이 낮아진다. 비전문가 집단(결정 역량 미보유)은 생성물의 품질을 객관적으로 판단하기 어려워 활용 여부를 합리적으로 결정하지 못한다. 결과적으로 두 집단 모두 적극적 채택 동기가 제한된다.
  • 책임 단계 (결과물 품질에 대한 책임): 전문가 집단은 AI 생성물 활용으로 인한 품질 리스크를 회피하기 위해 활용 자체를 제한한다. 비전문가 집단은 활용 이후 결과물에 대한 책임을 스스로 부담하거나, 실패 경험 이후 도구 사용을 중단하는 경향을 보인다. 이로 인해 서비스에 대한 신뢰도 및 지속 활용성이 저하된다.
▶2. 시스템 설계 및 해결방안System Design

a. 개요

현재 시장에는 이미 다양한 고품질 사운드 에셋이 축적되어 있으며, Krotos Studio와 같은 전문 툴은 프리셋과 알고리즘 기반 구조를 통해 높은 완성도의 사운드 디자인 에셋을 제공하고 있다. 이러한 도구들은 AI 기반 생성 기술이 목표로 하는 기능을 이미 안정적인 품질로 구현하고 있으나, 일정 수준의 전문성과 학습 비용이 요구된다는 진입장벽이 존재한다.

본 프로젝트는 다음 구조를 지향한다:

  • 재료 수급 & 사운드 에디팅 단계: AI 의존도를 최소화하고, 검증된 사운드 에셋과 프리셋 기반 구조를 중심으로 설계한다. 생성 품질의 불확실성을 제거하고 안정적인 결과물을 확보한다.
  • 싱크로나이징 단계: Claude API를 활용해 씬, 이벤트, 타이밍 정보를 자동 추출하고, 이를 기반으로 사운드 매칭 및 배치를 수행한다.
  • 플랫폼 제공: 웹 기반 SaaS가 아닌 미디어 편집 워크플로우에 직접 통합되는 범용 플러그인 형태로 설계하여, 사용자가 실무 환경 내에서 즉시 활용할 수 있도록 한다.

이를 통해 '생성 중심 구조'가 아닌 '분석·매칭·배치 중심 구조'를 기반으로 한 실무 친화형 자동화 플랫폼을 구현한다.

b. 시스템 구조도

HTML 구동 테스트 환경의 전체 흐름은 다음과 같다:

Video Input → Video Analysis (Claude API) → Event Timeline (JSON)
                                            → Asset Match (JSON)
                                            → Auto Placement
  • 1. 영상 입력: 사용자가 영상 파일을 업로드한다.
  • 2. 영상 분석: Claude API + Prompt를 통해 영상을 분석한다.
  • 3. Event Timeline: JSON 형태로 타임라인 정보를 출력한다.
  • 4. Asset Match: JSON 형태로 사운드 이벤트 정보를 매칭한다.
  • 5. Auto Placement: 전달된 정보를 기반으로 사운드 에셋을 배치한다.

c. 핵심 기술 — 영상 분석

영상 분석의 목적은 "씬 커버(Scene Coverage)" 관점에서 필요한 사운드 요소를 식별하고, 이를 배치 가능한 이벤트 타임라인으로 정규화하는 것이다. 분석 결과는 JSON으로 출력하며 각 이벤트는 다음을 포함한다:

  • 타임라인 정보: ms 단위 타임코드, fps 및 프레임 단위 정보 병기 가능
  • 사운드 카테고리: SFX / Foley / Ambience
  • 사운드 재질 (Material): 메탈, 돌, 풀, 유리 등
  • 사운드 무게 (Weight): light / medium / heavy
  • 사운드 동작 (Action): idle, move, hit, swing 등

영상 분석 룰은 에셋 라이브러리와의 매칭 정확도를 극대화하는 방향으로 설계하며, 추가 규칙을 포함한다:

  • 카테고리별 디폴트 레벨(dBFS) 제안값 출력
  • 동시 이벤트 발생 시 카테고리 기반 사이드체인(ducking) 규칙 적용

d. 핵심 기술 — 에셋 구성

에셋 라이브러리는 카테고리 / 재질 / 무게 / 동작 기준으로 설계한다. 에셋 메타데이터는 다음 기준을 포함한다:

  • 카테고리 (기본): SFX / Foley / Ambience
  • 카테고리 (상세, SFX):
    • Hard EFX — 씬 내 직접적 효과음 (충돌, 폭발, 타격 등)
    • SPFX — 씬 외/전환/연출 중심 효과음 (분위기 형성, 화면 전환, 특수 연출 등)
  • 사운드 재질 (Material): 사운드의 톤 기반 (메탈, 돌, 흙, 나무 등)
  • 사운드 무게 (Weight): 대상의 크기 및 저역 비중
  • 사운드 동작 (Action): Idle, Attack, Swing 등
▶3. 진행내역 — 실험 로그Experiment Log

사운드 자동화 파이프라인의 첫 검증 대상으로 Foley — 특히 Footstep 싱크를 선택했다. Foley는 영상 내 물리적 움직임과 1:1로 대응되는 사운드로, 타이밍 정확도가 가장 명확하게 검증 가능하다. 발소리 한 개가 맞고 틀리는지를 수치로 측정할 수 있어 AI 분석 능력의 실무 적용 가능성을 판단하기에 가장 적합한 케이스였다. 결과적으로 이 과정에서 AI의 구조적 한계와 강점이 동시에 드러났으며, 이는 이후 방향 설정의 근거가 됐다.

시도 1 — 픽셀 차분 (Frame Differencing) 실패

방법: 발 ROI를 고정 좌표로 지정하고 프레임 간 픽셀 변화량을 측정해 접지 순간을 감지하는 방식. 정밀도 향상을 위해 60fps 추출도 시도했으나 동일한 결과.

실패 원인: 카메라 무빙 발생 시 발이 지정된 ROI 밖으로 이탈하며 배경 변화나 컷 전환이 접지로 오탐된다. 이는 정밀도 문제가 아니라 근본적인 ROI 추적 구조의 한계다.

시도 2 — Pose Estimation (MediaPipe) 부분 성공, 구조적 한계 확인

방법: 발목 키포인트 Y좌표의 로컬 최저점을 접지 순간으로 판단하는 방식. 장고 클립(confidence 0.3 기준, 유효 프레임 63.9%, 접지 후보 30개)과 윈터솔져 클립(유효 23.9%, 액션씬 카메라 무빙으로 불안정)에서 테스트.

PE 감지       GT(정답)     오차
6933ms   →   6923ms     +10ms   ✓
7783ms   →   7762ms     +21ms   ✓
7216ms   →   7762ms    -546ms   ✗
8100ms   →   7762ms    +338ms   ✗

핵심 발견: 접지 하나당 후보가 여러 개 출력되는 노이즈 문제와 함께, 보다 근본적인 구조적 차이가 확인됐다. HAND·CLOTH는 "모션의 존재(existence)"를 감지하는 문제인 반면, Footstep은 "모션의 종료 순간(motion termination)"을 감지하는 문제다 — 이 둘은 본질적으로 다른 구조다. 발목 직접 추적 방식으로는 한계가 명확했다.

시도 3 — STATIC vs WALKING 디폴트 전략 변경 (v2.3 → 롤백) 롤백

방법: 씬 초반 정적 판단 오류를 해결하기 위해 프롬프트 디폴트를 walking으로 설정.

실패 원인: 실제 테스트에서 초반 누락 문제가 해결되지 않았고, 무조건 walking 가정은 실제 정적 씬에서 과도한 Footstep 출력을 유발했다. v2.301에서 롤백.

시도 4 — 멀티 신호 패턴 분석 (프롬프트 고도화) 부분 적용

방법: 단일 신호 의존에서 벗어나 복수 신호 교차검증 구조 설계.

  • HEAD_DIP: 머리/모자의 수직 진동 — 카메라 트래킹 중 Y좌표 비교 불안정
  • SHOULDER_OSCILLATION: 어깨 라인의 좌우·수직 진동
  • TORSO_SWAY: 셔츠 V라인 좌우 진동 — 정면샷에서만 유효

한계: 앵글과 조명 조건에 따라 신호 가용성이 크게 달라지며, armed/crouching/carrying 등 자세 변형 시 분류 기준이 복잡해진다. 특정 앵글·저조도 환경에서는 픽셀 정보 자체가 부족해진다.

시도 5 — GAIT_MODIFIER 도입 도입, 오용 문제 발생

방법: 무기 조준, 크라우칭, 하중 이동 등 자세 변형 상태에서 HEAD_DIP 신호 가중치를 낮추는 GAIT_MODIFIER 규칙 추가.

발생한 오용 패턴:

프롬프트 의도: "armed_stance → HEAD_DIP 가중치 줄여라"
API 실제 동작: "armed_stance → 아마 안 걷고 있겠지 → static 처리"

신호 약함과 이동 없음을 혼동. GAIT_MODIFIER는 신호 가중치 조정 전용이었으나 API가 이를 이동 여부 판단에 잘못 적용했다. v2.303에서 "신호 약함 ≠ 이동 없음" 규칙 명시적으로 추가해 수정.

시도 6 — ffmpeg + Claude API 파이프라인 구축 구조 완성, 근본 한계 재확인

방법: ffmpeg 기반 파이프라인과 Claude API를 결합한 자동화 구조 구축.

  • 4fps 스캔 → 전체 씬 파악 및 보행 구간 감지
  • 보행 구간만 30fps 추출 + 프레임별 타임스탬프 부여
  • Claude API에 "이 180장을 순서대로 봐, 각 장의 시간은 이거야" 형태로 전송
  • JSON 결과 반환

근본 한계 확인: Claude는 이미지 90장을 받아도 "연속된 영상"이 아닌 개별 이미지로 처리한다. 어깨 Y좌표가 프레임 5에서 얼마, 프레임 6에서 얼마 — 이 수치를 비교해 로컬 미니멈을 계산하는 것이 불가능하다. 결과적으로 "걷고 있으니까 발소리 있겠지"로 처리하는 방식으로 fallback된다.

시도 7 — OpenCV 3신호 수치 추적 동일한 벽 재확인

방법: AI에서 타이밍 판단을 분리하고, OpenCV로 3신호를 동시 수치 측정 후 로컬 미니멈으로 접지를 감지하는 구조. Claude는 분류만 담당.

  • 어깨 Y — 측면샷에서 유효
  • 어깨 X — 정면 트래킹샷에서 유효, 카메라 줌인 영향 없음
  • 머리 Y — 보조 신호

MIN_PEAK_DIST (8, 24, 30), MIN_PEAK_PROM (1.0, 2.0, 3.0, 5.0) 등 파라미터 조정 시도.

결론: 어떤 파라미터 조합에서도 같은 한계에 부딪혔다. MediaPipe 어깨/머리 추적으로 발 접지 타이밍을 잡는 것은 "Motion existence는 감지하지만 Motion termination moment는 감지하지 못한다"는 시도 2에서 도출한 결론과 정확히 동일한 벽이다. 다른 방향에서 접근했지만 같은 구조적 한계로 귀결됐다.

→ 피벗: Foley 정밀 싱크 → Ambience 파이프라인

Ambience는 "언제"가 아니라 "무엇이 보이는가"의 문제다. 씬 분류, 공간 크기, 환경 특성 파악에서 Claude API는 일관되게 실무 수준의 결과를 냈다. AI가 잘 풀 수 있는 구조의 문제부터 완성도를 높이는 방향으로 전환.
